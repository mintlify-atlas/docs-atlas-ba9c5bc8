Extensions let you add capabilities to Kuzu that are not part of the core engine. Each extension is a self-contained module that registers new functions, indexes, or data source connectors when loaded.

## Pre-installed extensions

Starting with **v0.11.3**, the following extensions ship with every Kuzu binary and do not require a separate download:

| Extension | Description |
|-----------|-------------|
| `algo`    | Graph algorithms: PageRank, weakly/strongly connected components, Louvain community detection, k-core decomposition, spanning forest |
| `fts`     | BM25-based full-text search indexing and querying across multiple string columns |
| `json`    | JSON parsing, construction, and manipulation functions |
| `vector`  | HNSW-based approximate nearest-neighbor vector similarity search |

Even though these extensions are bundled, you still need to load them before use:

```cypher
LOAD EXTENSION fts;
LOAD EXTENSION vector;
LOAD EXTENSION algo;
LOAD EXTENSION json;
```

## Manually installed extensions

Extensions not bundled with the binary must be installed from an extension repository before loading. The official Kuzu extension repository requires network access. For air-gapped environments, you can run a local mirror.

### Installing from the official repository

```cypher
INSTALL httpfs;
LOAD EXTENSION httpfs;
```

### Installing with a custom repository URL

Use the `FROM` clause to specify the repository host:

```cypher
INSTALL fts FROM 'http://localhost:8080/';
LOAD EXTENSION fts;
```

To force a reinstall of an already-installed extension, prefix with `FORCE`:

```cypher
FORCE INSTALL fts FROM 'http://localhost:8080/';
```

### Setting up a local extension server

For offline environments or reproducible builds, run the official extension repository Docker image:

<Steps>
  <Step title="Pull the image">
    ```bash
    docker pull ghcr.io/kuzudb/extension-repo:latest
    ```
  </Step>
  <Step title="Run the server">
    ```bash
    docker run -p 8080:80 ghcr.io/kuzudb/extension-repo:latest
    ```
  </Step>
  <Step title="Install and load an extension">
    ```cypher
    INSTALL fts FROM 'http://localhost:8080/';
    LOAD EXTENSION fts;
    ```
  </Step>
</Steps>

## Inspecting loaded extensions

To see which extensions are currently loaded in the active session:

```cypher
CALL SHOW_LOADED_EXTENSIONS() RETURN *;
```

To list all indexes created by loaded extensions:

```cypher
CALL SHOW_INDEXES() RETURN *;
```
