## Installation

<Tabs>
  <Tab title="cargo add">
    ```bash
    cargo add kuzu
    ```
  </Tab>
  <Tab title="Cargo.toml">
    ```toml
    [dependencies]

    ```
  </Tab>

<Note>
  By default, the Kuzu C++ library is compiled from source and statically linked into your binary. See [build configuration](#build-configuration) if you want to link against a pre-built library instead.
</Note>

## Quickstart

```rust
use kuzu::{Connection, Database, SystemConfig};

fn main() -> anyhow::Result<()> {
    let db = Database::new("./my_graph", SystemConfig::default())?;
    let conn = Connection::new(&db)?;

    conn.query("CREATE NODE TABLE Person(name STRING, age INT64, PRIMARY KEY(name));")?;
    conn.query("CREATE (:Person {name: 'Alice', age: 25});")?;
    conn.query("CREATE (:Person {name: 'Bob', age: 30});")?;

    let result = conn.query("MATCH (p:Person) RETURN p.name AS NAME, p.age AS AGE;")?;
    println!("{result}");

    Ok(())
}
```

## Database

`kuzu::Database` manages all database components. It is `Send + Sync` and safe to share across threads.

```rust
pub struct Database { /* ... */ }

impl Database {
    pub fn new<P: AsRef<Path>>(path: P, config: SystemConfig) -> Result<Self, Error>;
    pub fn in_memory(config: SystemConfig) -> Result<Self, Error>;
}
```

<ParamField path="path" type="P: AsRef<Path>">
  Path to the database directory. Pass `""` or `":memory:"` for an in-memory database.
</ParamField>

<ParamField path="config" type="SystemConfig">
  Database configuration. Use `SystemConfig::default()` for sensible defaults.
</ParamField>

### In-memory database

```rust
let db = Database::in_memory(SystemConfig::default())?;
// Equivalent to:
let db = Database::new(":memory:", SystemConfig::default())?;
```

## SystemConfig

`SystemConfig` uses a builder pattern. Call `SystemConfig::default()` and chain the methods you need to override.

```rust
pub struct SystemConfig {
    buffer_pool_size: u64,      // default: auto-detect
    max_num_threads: u64,       // default: auto-detect (CPU cores)
    enable_compression: bool,   // default: true
    read_only: bool,            // default: false
    max_db_size: u64,           // default: u32::MAX (temporary limit)
    auto_checkpoint: bool,      // default: true
    checkpoint_threshold: i64,  // default: -1 (library default = 16MB)
    throw_on_wal_replay_failure: bool,  // default: true
    enable_checksums: bool,     // default: true
}
```

<AccordionGroup>
  <Accordion title="buffer_pool_size(size: u64)">
    Maximum buffer pool size in bytes. Larger values reduce file I/O by keeping more pages in memory.
  </Accordion>
  <Accordion title="max_num_threads(n: u64)">
    Maximum number of threads for query execution. `0` auto-detects from CPU cores.
  </Accordion>
  <Accordion title="enable_compression(b: bool)">
    Enable on-disk compression. Defaults to `true`.
  </Accordion>
  <Accordion title="read_only(b: bool)">
    Open the database read-only. Multiple read-only `Database` values can share the same path.
  </Accordion>
  <Accordion title="max_db_size(size: u64)">
    Maximum database size in bytes. This is a temporary limit; the default is `u32::MAX`.
  </Accordion>
  <Accordion title="auto_checkpoint(b: bool)">
    Automatically checkpoint when the WAL file grows beyond `checkpoint_threshold`.
  </Accordion>
  <Accordion title="checkpoint_threshold(n: i64)">
    WAL file size in bytes that triggers a checkpoint. `-1` uses the library default (16 MB).
  </Accordion>
  <Accordion title="throw_on_wal_replay_failure(b: bool)">
    Return an error if WAL replay fails on startup. When `false`, silently replays as far as possible.
  </Accordion>
  <Accordion title="enable_checksums(b: bool)">
    Use checksums to detect WAL file corruption.
  </Accordion>
</AccordionGroup>

```rust
use kuzu::{Database, SystemConfig};

let config = SystemConfig::default()
    .buffer_pool_size(2 * 1024 * 1024 * 1024)  // 2 GB
    .max_num_threads(8)
    .read_only(false)
    .enable_compression(true);

let db = Database::new("./my_graph", config)?;
```

## Connection

`kuzu::Connection` is used to execute queries. It is `Send + Sync` and can be shared across threads.

```rust
pub struct Connection<'a> { /* ... */ }

impl<'a> Connection<'a> {
    pub fn new(database: &'a Database) -> Result<Self, Error>;

    pub fn query(&self, query: &str) -> Result<QueryResult<'a>, Error>;

    pub fn prepare(&self, query: &str) -> Result<PreparedStatement, Error>;

    pub fn execute(
        &self,
        prepared_statement: &mut PreparedStatement,
        params: Vec<(&str, Value)>,
    ) -> Result<QueryResult<'a>, Error>;

    pub fn set_max_num_threads_for_exec(&mut self, num_threads: u64);
    pub fn get_max_num_threads_for_exec(&self) -> u64;
    pub fn set_query_timeout(&self, timeout_ms: u64);
    pub fn interrupt(&self) -> Result<(), Error>;
}
```

The lifetime `'a` ties the connection to the database — connections cannot outlive their `Database`. Use [`std::thread::scope`](https://doc.rust-lang.org/std/thread/fn.scope.html) to create connections in worker threads:

```rust
std::thread::scope(|s| {
    let handle = s.spawn(|| {
        let conn = Connection::new(&db)?;
        conn.query("MATCH (p:Person) RETURN p.name")
    });
    handle.join().unwrap()
})?;
```

<Warning>
  Write queries must be executed sequentially. Kuzu returns `Error::FailedQuery` if a write query is attempted while another write is already running on a different connection.
</Warning>

## QueryResult

`QueryResult` implements `Iterator<Item = Vec<Value>>`. Each call to `next()` returns the columns of one row as a `Vec<Value>`.

```rust
pub struct QueryResult<'a> { /* ... */ }

impl<'a> QueryResult<'a> {
    pub fn get_num_columns(&self) -> usize;
    pub fn get_num_tuples(&self) -> u64;
    pub fn get_column_names(&self) -> Vec<String>;
    pub fn get_column_data_types(&self) -> Vec<LogicalType>;
    pub fn get_compiling_time(&self) -> f64;    // milliseconds
    pub fn get_execution_time(&self) -> f64;    // milliseconds

    // Arrow feature only
    #[cfg(feature = "arrow")]
    pub fn iter_arrow(&mut self, chunk_size: usize) -> Result<ArrowIterator, Error>;
}
```

### Iterating rows

```rust
let mut result = conn.query("MATCH (p:Person) RETURN p.name AS NAME, p.age AS AGE;")?;

for row in &mut result {
    // row: Vec<Value>
    println!("{:?}", row);
}
```

### Pattern matching on `Value`

```rust
use kuzu::Value;

for row in &mut result {
    if let Value::String(name) = &row[0] {
        if let Value::Int64(age) = &row[1] {
            println!("{name} is {age} years old");
        }
    }
}
```

### Display

`QueryResult` implements `Display`, which prints the result as a pipe-delimited table:

```rust
println!("{result}");
// NAME|AGE
// Alice|25
// Bob|30
```

## PreparedStatement

```rust
pub struct PreparedStatement { /* ... */ }
```

Prepare once, execute many times. Parameters are passed as `Vec<(&str, Value)>`.

```rust
conn.query("CREATE NODE TABLE Person(name STRING, age INT64, PRIMARY KEY(name));")?;

let mut stmt = conn.prepare("CREATE (:Person {name: $name, age: $age});")?;

for (name, age) in [("Alice", 25i64), ("Bob", 30), ("Carol", 22)] {
    conn.execute(&mut stmt, vec![
        ("name", Value::String(name.to_string())),
        ("age",  Value::Int64(age)),
    ])?;
}
```

## Value enum

`kuzu::Value` covers all types that Kuzu can return or accept as a query parameter.

```rust
pub enum Value {
    Null(LogicalType),
    Bool(bool),
    Int8(i8), Int16(i16), Int32(i32), Int64(i64),
    UInt8(u8), UInt16(u16), UInt32(u32), UInt64(u64),
    Float(f32), Double(f64),
    Date(time::Date),
    Interval(time::Duration),
    Timestamp(time::OffsetDateTime),
    String(String),
    Blob(Vec<u8>),
    Uuid(uuid::Uuid),
    List(LogicalType, Vec<Value>),
    Map(LogicalType, Vec<(Value, Value)>),
    Struct(Vec<(String, Value)>),
    Node(NodeVal),
    Rel(RelVal),
    InternalID(InternalID),
    // ...
}
```

Primitive Rust types convert into `Value` automatically via `From`:

```rust
let v: Value = 42i64.into();         // Value::Int64(42)
let v: Value = "hello".into();       // Value::String("hello".to_string())
let v: Value = true.into();          // Value::Bool(true)
```

## Error handling

```rust
pub enum Error {
    FailedQuery(String),
    FailedPreparedStatement(String),
    // ...
}
```

All fallible functions return `Result<T, kuzu::Error>`. Use the `?` operator or `match` for error handling:

```rust
match conn.query("MATCH (p:Person RETURN p.name") {  // syntax error
    Ok(_) => println!("ok"),
    Err(e) => eprintln!("Query failed: {e}"),
}
```

## Arrow integration

Enable the `arrow` Cargo feature for Apache Arrow output:

```toml
[dependencies]
kuzu = { version = "0.11.1", features = ["arrow"] }
```

```rust
let mut result = conn.query("MATCH (p:Person) RETURN p.name, p.age")?;

for batch in result.iter_arrow(1000)? {
    // batch: arrow::record_batch::RecordBatch
    println!("{} rows", batch.num_rows());
}
```

## Build configuration

By default, the C++ library is compiled and statically linked. Override this with environment variables:

```bash
# Link against a pre-built library dynamically
export KUZU_SHARED=1
export KUZU_INCLUDE_DIR=/path/to/kuzu/include
export KUZU_LIBRARY_DIR=/path/to/kuzu/lib

cargo build
```

On macOS with Homebrew:

```bash
brew install kuzu
export KUZU_SHARED=1
export KUZU_LIBRARY_DIR=$(brew --prefix)/lib
export KUZU_INCLUDE_DIR=$(brew --prefix)/include
```

### Using extensions in binaries

Kuzu extensions use dynamic symbol resolution. Add `--export-dynamic` (or `-rdynamic`) to your linker flags, otherwise extensions will fail with undefined-symbol errors at runtime. Add to `build.rs`:

```rust
fn main() {
    println!("cargo:rustc-link-arg=-rdynamic");
}
```

## Full example

```rust
use kuzu::{Connection, Database, SystemConfig, Value};

fn main() -> anyhow::Result<()> {
    let db = Database::new("./social_graph", SystemConfig::default())?;
    let conn = Connection::new(&db)?;

    // Schema
    conn.query("CREATE NODE TABLE Person(name STRING, age INT64, PRIMARY KEY(name));")?;
    conn.query("CREATE REL TABLE Knows(FROM Person TO Person, since INT64);")?;

    // Data
    conn.query("CREATE (:Person {name: 'Alice', age: 30});")?;
    conn.query("CREATE (:Person {name: 'Bob', age: 25});")?;
    conn.query(
        "MATCH (a:Person {name: 'Alice'}), (b:Person {name: 'Bob'}) \
         CREATE (a)-[:Knows {since: 2020}]->(b);"
    )?;

    // Prepared statement
    let mut stmt = conn.prepare(
        "MATCH (a:Person {name: $name})-[:Knows]->(b) RETURN b.name, b.age;"
    )?;
    let mut result = conn.execute(&mut stmt, vec![
        ("name", Value::String("Alice".to_string())),
    ])?;

    for row in &mut result {
        println!("{:?}", row);
    }

    // Metadata
    println!("Compiled in {:.2}ms, executed in {:.2}ms",
        result.get_compiling_time(),
        result.get_execution_time(),
    );

    Ok(())
}
```
