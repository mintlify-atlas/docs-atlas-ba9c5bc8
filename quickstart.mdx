This guide walks you through creating a small social graph using the Python API. You will define node and relationship tables, load data, and query the graph with Cypher.

<Note>
  Make sure you have Python 3.8 or higher installed. If you haven't installed Kuzu yet, see the [installation guide](/installation).
</Note>

## Full working example

The example below creates a social graph with `User` and `City` nodes connected by `Follows` and `LivesIn` relationships. This is the same schema used in Kuzu's [demo dataset](https://github.com/kuzudb/kuzu/tree/master/dataset/demo-db/csv).

```python
import kuzu

# Open a persistent database (or use ":memory:" for in-memory)
db = kuzu.Database("./demo_db")
conn = kuzu.Connection(db)

# Define the schema
conn.execute("CREATE NODE TABLE User(name STRING, age INT64, PRIMARY KEY (name))")
conn.execute("CREATE NODE TABLE City(name STRING, population INT64, PRIMARY KEY (name))")
conn.execute("CREATE REL TABLE Follows(FROM User TO User, since INT64)")
conn.execute("CREATE REL TABLE LivesIn(FROM User TO City)")

# Insert nodes
conn.execute("CREATE (:User {name: 'Adam',    age: 30})")
conn.execute("CREATE (:User {name: 'Karissa', age: 40})")
conn.execute("CREATE (:User {name: 'Zhang',   age: 50})")
conn.execute("CREATE (:User {name: 'Noura',   age: 25})")
conn.execute("CREATE (:City {name: 'Waterloo',  population: 150000})")
conn.execute("CREATE (:City {name: 'Kitchener', population: 250000})")
conn.execute("CREATE (:City {name: 'Guelph',    population: 130000})")

# Insert relationships
conn.execute("MATCH (a:User {name: 'Adam'}),    (b:User {name: 'Karissa'}) CREATE (a)-[:Follows {since: 2020}]->(b)")
conn.execute("MATCH (a:User {name: 'Adam'}),    (b:User {name: 'Zhang'})   CREATE (a)-[:Follows {since: 2020}]->(b)")
conn.execute("MATCH (a:User {name: 'Karissa'}), (b:User {name: 'Zhang'})   CREATE (a)-[:Follows {since: 2021}]->(b)")
conn.execute("MATCH (a:User {name: 'Zhang'}),   (b:User {name: 'Noura'})   CREATE (a)-[:Follows {since: 2022}]->(b)")
conn.execute("MATCH (u:User {name: 'Adam'}),    (c:City {name: 'Waterloo'})  CREATE (u)-[:LivesIn]->(c)")
conn.execute("MATCH (u:User {name: 'Karissa'}), (c:City {name: 'Waterloo'})  CREATE (u)-[:LivesIn]->(c)")
conn.execute("MATCH (u:User {name: 'Zhang'}),   (c:City {name: 'Kitchener'}) CREATE (u)-[:LivesIn]->(c)")
conn.execute("MATCH (u:User {name: 'Noura'}),   (c:City {name: 'Guelph'})    CREATE (u)-[:LivesIn]->(c)")

# Query: who does Adam follow?
result = conn.execute(
    "MATCH (a:User {name: 'Adam'})-[:Follows]->(b:User) RETURN b.name, b.age"
)
while result.has_next():
    print(result.get_next())
# ['Karissa', 40]
# ['Zhang', 50]
```

## Step-by-step walkthrough

<Steps>
  <Step title="Install Kuzu">
    Install the `kuzu` package from PyPI using pip.

    ```bash
    pip install kuzu
    ```

    Kuzu requires **Python 3.8 or higher**. No additional system dependencies are needed — the native library is bundled in the wheel.
  </Step>

  <Step title="Create a database and connection">
    Import `kuzu` and open a database. Pass a directory path to store the database on disk, or use `":memory:"` (or `None`) for a temporary in-memory database.

    ```python
    import kuzu

    # Persistent database stored at ./demo_db
    db = kuzu.Database("./demo_db")

    # In-memory database (data is lost when the process exits)
    # db = kuzu.Database(":memory:")

    conn = kuzu.Connection(db)
    ```

    <Tip>
      Use the database as a context manager to ensure it is closed cleanly even if an exception occurs.

      ```python
      with kuzu.Database("./demo_db") as db:
          conn = kuzu.Connection(db)
          # ... queries
      ```
    </Tip>
  </Step>

  <Step title="Define the schema">
    Create node tables and relationship tables using Cypher DDL statements. Each node table must declare a primary key.

    ```python
    # Node tables
    conn.execute("CREATE NODE TABLE User(name STRING, age INT64, PRIMARY KEY (name))")
    conn.execute("CREATE NODE TABLE City(name STRING, population INT64, PRIMARY KEY (name))")

    # Relationship tables
    conn.execute("CREATE REL TABLE Follows(FROM User TO User, since INT64)")
    conn.execute("CREATE REL TABLE LivesIn(FROM User TO City)")
    ```

    `Follows` is a relationship from `User` to `User` with a `since` property. `LivesIn` is a relationship from `User` to `City` with no additional properties.
  </Step>

  <Step title="Load data">
    You can insert data row-by-row with `CREATE`, or bulk-load from CSV files using `COPY ... FROM`.

    <Tabs>
      <Tab title="CREATE statements">
        ```python
        conn.execute("CREATE (:User {name: 'Adam',    age: 30})")
        conn.execute("CREATE (:User {name: 'Karissa', age: 40})")
        conn.execute("CREATE (:City {name: 'Waterloo', population: 150000})")

        conn.execute(
            "MATCH (a:User {name: 'Adam'}), (b:User {name: 'Karissa'}) "
            "CREATE (a)-[:Follows {since: 2020}]->(b)"
        )
        conn.execute(
            "MATCH (u:User {name: 'Adam'}), (c:City {name: 'Waterloo'}) "
            "CREATE (u)-[:LivesIn]->(c)"
        )
        ```
      </Tab>
      <Tab title="COPY FROM CSV">
        ```python
        # Bulk-load from CSV files (fastest for large datasets)
        conn.execute('COPY User FROM "user.csv"')
        conn.execute('COPY City FROM "city.csv"')
        conn.execute('COPY Follows FROM "follows.csv"')
        conn.execute('COPY LivesIn FROM "lives-in.csv"')
        ```

        CSV files for the demo dataset are available in the [Kuzu repository](https://github.com/kuzudb/kuzu/tree/master/dataset/demo-db/csv).
      </Tab>
    </Tabs>
  </Step>

  <Step title="Run queries">
    Use `conn.execute()` to run Cypher queries. The method returns a `QueryResult` object.

    ```cypher
    -- Find all users and their ages
    MATCH (u:User) RETURN u.name, u.age;

    -- Find who Adam follows
    MATCH (a:User {name: 'Adam'})-[:Follows]->(b:User)
    RETURN b.name, b.age;

    -- Find users and the cities they live in
    MATCH (u:User)-[:LivesIn]->(c:City)
    RETURN u.name, c.name;

    -- Two-hop traversal: find friends-of-friends
    MATCH (a:User {name: 'Adam'})-[:Follows]->(b:User)-[:Follows]->(c:User)
    RETURN a.name, b.name, c.name;
    ```

    ```python
    result = conn.execute(
        "MATCH (u:User)-[:LivesIn]->(c:City) RETURN u.name, c.name"
    )
    while result.has_next():
        print(result.get_next())
    # ['Adam', 'Waterloo']
    # ['Karissa', 'Waterloo']
    # ['Zhang', 'Kitchener']
    # ['Noura', 'Guelph']
    ```
  </Step>

  <Step title="Iterate results and convert to DataFrame">
    `QueryResult` is iterable. Use `has_next()` and `get_next()` to read rows one at a time, or call `get_as_df()` to convert the entire result to a pandas DataFrame.

    ```python
    # Iterate row by row
    result = conn.execute("MATCH (u:User) RETURN u.name, u.age")
    while result.has_next():
        row = result.get_next()  # returns a list: ['Adam', 30]
        print(row)

    # Convert to a pandas DataFrame
    result = conn.execute("MATCH (u:User) RETURN u.name, u.age ORDER BY u.age")
    df = result.get_as_df()
    print(df)
    #      u.name  u.age
    # 0     Noura     25
    # 1      Adam     30
    # 2   Karissa     40
    # 3     Zhang     50

    # Convert to a Polars DataFrame
    result = conn.execute("MATCH (u:User) RETURN u.name, u.age")
    pl_df = result.get_as_pl()

    # Convert to a PyArrow Table
    result = conn.execute("MATCH (u:User) RETURN u.name, u.age")
    arrow_table = result.get_as_arrow()
    ```

    <Tip>
      `QueryResult` also supports the Python iterator protocol, so you can use it directly in a `for` loop.

      ```python
      for row in conn.execute("MATCH (u:User) RETURN u.name, u.age"):
          print(row)
      ```
    </Tip>
  </Step>
</Steps>

## In-memory mode

Pass `":memory:"` or `None` as the database path to create a database that lives only in RAM. In-memory databases are useful for testing and prototyping — no files are written, and all data is lost when the process exits.

```python
import kuzu

db = kuzu.Database(":memory:")
conn = kuzu.Connection(db)

conn.execute("CREATE NODE TABLE Person(name STRING, age INT64, PRIMARY KEY (name))")
conn.execute("CREATE (:Person {name: 'Alice', age: 25})")

result = conn.execute("MATCH (p:Person) RETURN p.name, p.age")
while result.has_next():
    print(result.get_next())
# ['Alice', 25]
```
